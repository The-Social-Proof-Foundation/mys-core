This page describes how to contribute to MySocial, and provides additional information about participating in the MySocial community.

## Join the community {#join-the-community}

To connect with the MySocial community, join our [Discord](https://discord.gg/mys).

## Open issues {#open-issues}

To report an issue with MySocial, [create an issue](https://github.com/MystenLabs/mys/issues/new/choose) in the GitHub repo. Click **Get started** to open a template for the type of issue to create.

## Fork MySocial to contribute {#fork-mys-contribute}

To contribute to MySocial source code or documentation, you need only a GitHub account. You can commit updates and then submit a PR directly from the Github website, or create a fork of the repo to your local environment and use your favorite tools to make changes. Always submit PRs to the `main` branch.

See [MySocial Environment Setup](./mys-environment.mdx) for instructions on forking the MySocial repository, if necessary.

## Contribute via the MySocial Improvement Proposal (SIP) process {#SIP}

The MySocial Network is an open-source, decentralized, and permissionless protocol that welcomes community contributions. If you have an idea regarding a core protocol upgrade, the best way to make your voice heard is via submitting a MySocial Improvement Proposal (SIP). For more information on SIPs, see [Contribute to MySocial through SIPs](https://blog.mys.io/mys-improvement-proposals-sips/).
